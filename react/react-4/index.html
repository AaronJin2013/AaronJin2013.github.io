基于React进行框架开发的理解(四) ---- ts-node

Veröffentlicht am 2016-06-22

环境

ts-node

由于使用了TS,ts-node将会取代node对代码进行解析,同时也会获取目录下的tsconfig和所有的.d.ts来进行编译

mongoose

在node中使用mongodb,最佳方式是使用mongoose进行连接

Webpack middleware

在node中使用devMiddleware和hotMiddleware,用来在Webpack配置开发模式

router

使用express中的Router功能

Models

在ts-node中,我们需要使用到类型定义的Interface,Mongoose的Schema以及空数据的Model格式,因此每个model都需要定义这3个部分

项目心得

数据定义

在ts-node下,大部分的编写与node下并无不同,只是前后端的数据都可以由TS来进行类型定义,因此一个统一的Model是很有必要的,只是Model的3个部分尚没有找到复用的方式

前端和后端通讯json或string需要通过headers设置

headers: {
‘Content-Type’: ‘application/json’
}
application/json可以传递json
x-www-form-urlencoded可以传递字符串
不可串用,否则后端获取不到数据
后端需要添加
app.use(bodyParser.json());
app.use(bodyParser.urlencoded({ extended: true }));
